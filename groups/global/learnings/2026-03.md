## 2026-03-23 - WebMCP: Every Website Is Getting an Agent (nekuda)
- Source: nekuda Substack (2026-03-23) — https://nekuda.substack.com/p/webmcp-every-website-is-getting-an
- WebMCP is a new web standard (Google + Microsoft) that lets websites declare their capabilities as structured tools — so any AI agent (Gemini side panel, Claude extension, etc.) can call them directly without DOM scraping
- Chrome's Gemini side panel is rolling out globally to 3.8B users — the browser becomes the agent interface for every website
- For merchants: WebMCP replaces the need to build a dedicated on-site agent (like Amazon Rufus) — implement the standard and any browser AI becomes your store's agent
- Key use cases: natural language product discovery, filtering, comparison, cart actions, support — all through the browser chat
- Google also announced Universal Commerce Protocol (UCP, co-developed with Shopify/Walmart/Target) enabling agentic checkout through the browser chat
- Main gap: no authentication — remote bots/agents can call WebMCP tools the same way real users do; merchants need to think about rate limiting
- Relevance to Fishbone:
  - Assumption #1 (catalog quality -> AI visibility): WebMCP exposes product catalog tools directly to AI agents — well-structured catalog = better tool responses = better AI-driven discovery. Catalog quality is now even more directly a lever.
  - Assumption #6 (ChatGPT/Gemini apps as retailer channel): Gemini in Chrome is the same pattern — AI layer on top of product data, with 3.8B user distribution. Retailers who instrument WebMCP get broad AI agent compatibility for free. The "browser as the agent" framing is a strong parallel to the ChatGPT app angle Fishbone is exploring with Stor.AI.
  - Nekuda (who wrote this) builds agentic commerce tools for retailers — they're worth watching as a potential signal source and possible overlap/competitor.
